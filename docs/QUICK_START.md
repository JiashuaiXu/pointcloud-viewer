# 快速启动指南

## 🚀 React 项目快速启动

### 1. 启动开发服务器

```bash
npm run dev:react
```

### 2. 查看输出

**正常输出应该类似：**
```
VITE v5.4.21  ready in XXXX ms

➜  Local:   http://localhost:8082/
➜  Network: use --host to expose
```

**注意：** 如果 8082 端口被占用，Vite 会自动使用下一个可用端口（如 8083, 8084）

### 3. 访问应用

打开浏览器访问终端显示的地址（通常是 http://localhost:8082）

### 4. 检查界面

**应该看到：**
- ✅ 深色背景（#1a1a1a）
- ✅ 左上角：信息面板（显示点数量、FPS）
- ✅ 右上角：控制面板（文件选择按钮、重置相机按钮）

### 5. 测试文件选择

1. 点击右上角"选择点云文件"按钮
2. 选择一个点云文件（.las, .laz, .ply, .pcd, .potree）
3. 观察控制台日志

## 🔍 如果页面空白

### 检查步骤：

1. **打开浏览器开发者工具（F12）**
2. **查看 Console 标签页**
3. **应该看到以下日志：**
   ```
   [main.tsx] 开始初始化 React 应用
   [main.tsx] 找到 #app 元素，开始渲染
   [main.tsx] React 应用已渲染
   [App] 组件渲染
   [PotreeViewer] 组件渲染
   [usePotree] useEffect 执行
   [PotreeManager] initViewer 开始
   ...
   ```

4. **如果没有日志或看到错误：**
   - 检查终端是否有编译错误
   - 检查浏览器控制台的错误信息
   - 确认访问的端口号正确

## ⚠️ 常见问题

### 问题 1: 404 错误

**原因：** 访问了错误的端口或路径

**解决：**
1. 查看终端输出的实际端口号
2. 使用正确的 URL（如 http://localhost:8082）
3. 确保路径是 `/` 而不是其他路径

### 问题 2: 页面空白

**检查：**
1. 浏览器控制台是否有错误
2. 网络请求是否成功（Network 标签页）
3. 终端是否有编译错误

### 问题 3: 看不到文件选择按钮

**检查：**
1. 右上角是否有控制面板
2. 控制面板是否被其他元素遮挡
3. 浏览器缩放比例是否正常
